# Data Analytic Right

> The Data Analytic Right cleaves the irreducible event of "data-to-insight mapping." Every API call, every model inference — recorded as an unforgeable event on the engine log.

## The Dual-Lock Structure

**Institutional Lock (L3):** Who may access the data interface? Data holders control access conditions through API standards — who, under what conditions, at what frequency.

**Technical Lock (L4):** Every access is logged. Who, when, what data, what output — all written to an immutable log. The technical lock does not prevent access — it **records access**. The record manufactures verifiability.

## Reputation Good

Unlike Spacetime Analytic Right (a usage good — each takeoff consumes a time window), Data Analytic Right is a **reputation good**. Each verified use adds a layer of auditable history. A dataset's analytic right called by 100 clients without dispute is more valuable than one never called — not because the data improved, but because the **verification history thickened**.

## Duality

| Dimension | Spacetime Analytic Right | Data Analytic Right |
|:---|:---|:---|
| Object | Physical airspace unit | Data-to-insight mapping event |
| Closure | On the slot pool | On the engine log |
| Asset type | Usage good (consumable) | Reputation good (cumulative) |
| Exclusivity | Scheduling system enforces | Engine log naturally creates |
| Verifiability | Flight path tracked by base stations | API calls logged |
| Transferability | Slot tokens on exchange | Analytic right tokens on exchange |

## Product Chain

Spot → Futures → Options → ETF/Index → ABS → Swaps. Six products sharing one engine log. Not six separate products — one analytic right in six financial structures.
